🎯 Цель проекта
• Какую задачу решает приложение: Визуализация движения планет Солнечной системы в реальном времени и на заданную дату.
• Почему эта задача важна: Помогает наглядно понять принципы орбитального движения планет и их взаимное расположение.
• Целевая аудитория: Студенты, школьники, любители астрономии.
________________________________________
📌 Требования к приложению
Функциональные требования
• Отображение Солнца и 8 планет.
• Анимация движения планет по орбитам.
• Установка положения планет по выбранной дате.
• Просмотр краткой информации о планете при нажатии.
• Сплэш-скрин при запуске.
Нефункциональные требования
• Производительность: Плавная отрисовка графики (60 FPS).
• Удобство использования: Интуитивно понятный интерфейс с тремя кнопками.
• Надёжность: Стабильная работа на версиях Android от Oreo (API 28) до Android 15 (API 35).

🧠 Общая архитектура
• Выбранный подход:
– Simple View-Controller (MVC)

• Почему выбран именно он: Проект является графически ориентированным, и прямое взаимодействие Activity с Custom View обеспечивает минимальные задержки и простоту реализации.

• Основные компоненты системы:
– Activity (Start, MainActivity)
– Custom View (SolarSystemView)

🧩 Подробное описание классов
📌 Класс: Start
Роль: Начальный экран (Splash Screen).
Ответственность: Создание приветственного интерфейса и переход к основному экрану через 2 секунды.
Основные методы: 
- onCreate() — инициализация экрана и запуск CountDownTimer.
Взаимодействие с другими классами: Запускает MainActivity через Intent.
________________________________________
📌 Класс: MainActivity
Роль: Главный контроллер интерфейса.
Ответственность: Обработка нажатий на кнопки (Старт, Стоп, Дата) и управление состоянием SolarSystemView.
Основные методы:
- onCreate() — привязка UI-компонентов и установка слушателей событий.
Взаимодействие с другими классами: Управляет методами SolarSystemView.
________________________________________
📌 Класс: SolarSystemView
Роль: Ядро визуализации.
Ответственность: Расчет координат планет на основе времени/даты и их отрисовка на Canvas. Обработка касаний для отображения информации.
Почему используется: Кастомная отрисовка необходима для динамического отображения орбитальных систем.
________________________________________
🔄 Диаграмма работы приложения
Сценарий:
1. Пользователь открывает приложение (Start Activity).
2. Через 2 секунды открывается MainActivity.
3. Пользователь нажимает "START", планеты начинают двигаться.
4. Пользователь нажимает на планету, появляется диалоговое окно с информацией.
5. Пользователь выбирает дату через "DATE", планеты перестраиваются.

🎨 UI/UX анализ
• Почему интерфейс сделан именно так: Минималистичный дизайн на черном фоне имитирует космическое пространство.
• Какие принципы использованы:
– Простота: Всего три кнопки управления.
– Логичность: Кнопки расположены снизу, не перекрывая графику.
– Доступность: Крупные элементы управления.
• Что можно улучшить: Добавить масштабирование (pinch-to-zoom) и более детальные текстуры планет.

⚙️ Работа с потоками
Опишите: - Используются ли:
- invalidate() loop в главном потоке для анимации.
• Почему выбран этот способ: Для простых 2D-анимаций метод invalidate() является стандартным и эффективным.
• Как предотвращаются:
– зависания (ANR): Расчеты тригонометрии выполняются быстро и не блокируют поток.
– утечки памяти: Используются стандартные жизненные циклы Activity.
________________________________________
💾 Работа с данными
• Где хранятся данные: В константах класса SolarSystemView (орбитальные периоды, радиусы).
• Почему выбран этот способ: Данные статичны и не требуют внешней базы данных.
• Как обеспечивается:
– сохранность: Данные зашиты в код.
– корректность: Используются астрономически обоснованные соотношения периодов.
